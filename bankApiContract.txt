               ADMIN FEATURES
1. Employee Management (Create, Update, Status Control)
Why

Banks require controlled access. Employees manage accounts, money, and customer data, so admins must be able to create or restrict their privileges.


Admin can:

Add new employees

Update employee information

Activate, suspend, or terminate accounts

Admin uses employee management endpoints. The system validates:

Unique email

Role must be EMPLOYEE

Status changes affect login permissions

Employee records are stored in the employee table with role-based access ties.

2. Customer Oversight (Status Management)
Why

Admins must override employee decisions for compliance, fraud detection, or legal cases.

Admin can change customer status to:

ACTIVE

SUSPENDED

BLOCKED

Admin sends a patch request to modify the customer status field.
Blocked customer cannot log in or perform transactions.

3. System-Wide Account Monitoring

Management needs visibility into bank performance and account activity.


Admin can view all accounts with:

Holder name

Balance

Status

Type


Admin requests aggregated account data.
Filters like active, frozen, high-balance accounts can be applied.

4. Freeze/Unfreeze Accounts


Required for investigations, fraud, suspicious activity, or regulatory requests.



Admin controls whether an account:

Is fully operational

Frozen (no debit/credit allowed)

Admin updates status on the account record.
System checks “FROZEN” status before processing any transaction.

5. High-Level Transaction Monitoring


Banks must comply with audit and anti-money-laundering regulations.


Admin can see:

All deposits

Withdrawals

Transfers

Suspicious flags


Data is fetched using transaction logs with filters for:

Date range

Amount

Employee who processed it

Customer

6. Loan Policy & Approval Management

Loan rules differ per bank and must be controlled centrally.



Admin can:

Set interest rates

Set maximum approval limits

Approve high-value loans (beyond employee authority)


Policies stored in a settings table.
Employee approval checks against these thresholds.

7. Reporting & Audit Logging


Management requires performance insights and auditing for compliance.



Admin gets:

Total customers

Total accounts

Total balance

Daily/monthly transactions

Compliance logs



Backend aggregates values from:

Customers

Accounts

Transaction logs

Loans

Audit table

                  EMPLOYEE FEATURES


1. Customer Onboarding


Employees create customer records when someone opens a bank account in a branch.


Employee collects:

Name

Contact details

Address

KYC documents


Employee submits the onboarding form.
System validates KYC and stores customer information.

2. Update Customer Profile


Customer details may change (address, phone, etc.).



Employee updates relevant fields to keep records accurate.



Employee sends update request to the customer service module.
Only allowed fields are updated, old data is logged.

3. Change Customer Status


Needed for:

KYC updates pending

Non-compliance

Temporary disabling



Employee can mark customer as ACTIVE or SUSPENDED.


Employee updates status in the customer table.
Suspended customers cannot transact.

4. Open Bank Account


Customers need accounts to deposit or withdraw money.


Employee opens:

Savings account

Current account


Employee selects customer → chooses type → enters initial deposit.
System generates a unique account number.

5. Account Closure Request


Customers may close accounts when not needed.



Employee submits a closure request with reason.



Account status changes to CLOSED.
Balance must be zero before closure.

6. Cash Deposit (Branch Transaction)


Customers often deposit money physically at branches.



Employee adds cash to the customer account.



Employee enters amount → system credits account → transaction logged.

7. Cash Withdrawal (Branch Transaction)


Cash withdrawal in-branch requires verification.



Employee allows debit after verifying identity.



Employee enters amount → system checks balance → debits account.

8. Loan Application Verification


Every loan requires document verification.



Employee verifies:

Income proof

ID proof

Banking history


Employee marks loan as "verified" with remarks.
Only verified applications go to approval stage.

9. Loan Approval / Rejection


Low-value loans can be approved at branch level.



Employee updates loan status:

APPROVED

REJECTED


Employee checks verification → updates status.
System records decision + remarks.

10. Card Issuance / Replacement


Customers request new debit/credit cards.



Employee initiates:

New card request

Replacement card


Employee selects account → enters card type → system generates card details and sends request to card-processing subsystem.
